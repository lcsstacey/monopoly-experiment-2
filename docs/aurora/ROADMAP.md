# Project Aurora Roadmap

## Phase 0 — Pre-production (Weeks 1-4)

- Finalize game pillars and risk register.
- Create technical architecture skeleton (client/server/shared).
- Implement movement, combat hit validation, and enemy AI prototype.
- Select backend hosting and observability stack.
- Write legal/IP checklist for "inspired by" development.

**Exit criteria**
- Playable graybox arena with 2-player co-op and synced combat.
- Architecture decision record (ADR) set approved.

## Phase 1 — Vertical Slice (Weeks 5-12)

- Social hub + party flow.
- Procedural dungeon generation (rooms + connectors + encounter budget).
- Progression prototype (XP, item drops, rarity).
- 3 weapon families fully playable.
- Dedicated server deployment in one region.
- Basic telemetry and live ops admin panel.

**Exit criteria**
- Full 15-25 minute run loop complete.
- 20 external testers complete at least one run each.

## Phase 2 — Closed Alpha (Months 4-6)

- Account system hardening, anti-cheat guardrails.
- Content expansion (biomes, enemies, bosses, gear set bonuses).
- Matchmaking + friend invites + reconnect flow.
- Steam integration prototypes (overlay, achievements, cloud saves where needed).

**Exit criteria**
- 200-500 invited testers.
- Crash-free session rate and latency targets met.

## Phase 3 — Steam Playtest (Months 7-10)

- Steam store page and community setup.
- Onboarding polish, tutorialization, and UX iteration.
- Economy balancing and drop table tuning.
- Live patch pipeline and rollback readiness.

**Exit criteria**
- Public playtest readiness.
- Launch checklist with release candidate quality gates.

## Phase 4 — Early Access Candidate

- Content cadence plan (6-8 week updates).
- Retention systems and social features.
- Monetization QA and regional compliance checks.
- Scale testing and cost optimization.
