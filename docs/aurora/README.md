# Project Aurora (working title)

A cooperative online action RPG inspired by the **feel** of session-based dungeon crawling, but with original lore, art direction, systems, and economy.

## Product direction

- **Genre:** Online co-op action RPG (session-based dungeon runs).
- **Target platform (phase 1):** Windows PC.
- **Long-term distribution:** Steam.
- **Game camera:** Top-down / slightly angled, readable combat space.
- **Core loop:**
  1. Form party in social hub.
  2. Queue into procedural dungeon run.
  3. Fight, loot, upgrade, extract.
  4. Return to hub for crafting and progression.

## Design pillars

1. **Readable combat first**
   - Low visual noise, clear telegraphs, high skill expression.
2. **Friends-first co-op**
   - Fast invite/join flow, low friction matchmaking, drop-in sessions.
3. **Short-session progress**
   - 15–30 minute runs with meaningful rewards.
4. **Build expression**
   - Distinct weapon families, armor perks, and tactical loadouts.
5. **Live-service sanity**
   - Operable by a small team: stable cadence, sustainable content pipeline.

## Non-negotiables

- Build an original IP (no reused assets, story, names, or proprietary content).
- Server-authoritative multiplayer for anti-cheat and progression integrity.
- Co-op baseline quality: reconnect, host migration strategy (or dedicated servers), robust state sync.

## First playable scope ("Dungeon Slice v0")

- 1 social hub map.
- 2 procedural dungeon biomes.
- 4-player online co-op.
- 3 weapon archetypes (Melee, Ranged, Utility/Explosive).
- 6 enemy archetypes + 2 minibosses.
- Basic loot tiers and crafting.
- Account progression up to level 10.

## What success looks like in first 90 days

- Local prototype reaches stable 4-player co-op runs.
- 1 full run from lobby -> dungeon -> rewards -> hub.
- Average session 20+ minutes without critical desync.
- Instrumentation dashboards for crashes, session success rate, and retention basics.
